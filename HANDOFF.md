# HANDOFF — ECB Stylez Shopify Theme

A complete technical overview for anyone picking up this project cold. This is the
custom Shopify theme for **Kassie / ECB Stylez** ([@theecbstylez](https://www.instagram.com/theecbstylez/)),
a hair stylist & braider.

---

## 1. Project overview

- **What it is:** A custom Shopify storefront theme for a single-stylist hair/braiding
  business. It presents services, a gallery (before/after), about/bio, salon policies,
  a contact form, and a native product/cart/checkout flow. Products, collections,
  orders, and inventory all stay native in Shopify admin; nearly all marketing content
  (services, prices, policies, hours, links, reviews) is editable in the Shopify
  **theme editor** rather than hardcoded.
- **Platform:** Shopify **Online Store 2.0** Liquid theme. Uses JSON templates +
  sections + section groups (the OS 2.0 architecture), so every page is section-based
  and merchant-editable.
- **Build/stack:** **No build step.** Plain CSS (`assets/base.css`) and vanilla JS
  (`assets/theme.js`) — no Node, no `package.json`, no SCSS, no bundler, no framework.
- **Theme identity:** `ECB Stylez`, version `1.0.0` (see `config/settings_schema.json`
  → `theme_info`).
- **Design language:** Silver & black, clean/premium, mobile-first. Single font family
  (**Inter**, loaded from Google Fonts in the layout). No paid apps required, though the
  store has booking app embeds wired up (see §7).
- **Deployment:** Intended to connect to a Shopify store via **Online Store → Themes →
  Add theme → Connect from GitHub**, or via Shopify CLI (`shopify theme dev` /
  `shopify theme push`). Once connected, code edits push from GitHub and the client edits
  content in the theme editor.
- **Reference docs in repo:** `README.md` (operator-facing setup checklist) and
  `BUILD_SPEC.md` (a ~100KB verbatim build spec / rebuild instructions). This HANDOFF
  is the technical map; those two are the source material.

  --color-text: {{ settings.color_text }};
  --color-muted: {{ settings.color_muted }};
  --color-silver: {{ settings.color_silver }};
  --color-border: {{ settings.color_border }};
  --color-button-text: {{ settings.color_button_text }};
  --font-heading: 'Inter', ...;     /* hardcoded */
  --font-body: 'Inter', ...;        /* hardcoded */
  --font-heading-weight: 600;       /* hardcoded */
  --page-width: 1200px;             /* hardcoded */
}

## 7. Custom features (beyond a stock/base theme)

This is a **fully custom theme**, not a Dawn derivative. Everything is purpose-built. The
notable custom logic:

1. **Service menu as content, not products** (`service-menu.liquid` + `service-row.liquid`).
   Services, prices, durations, and per-service booking links are theme-editor blocks —
   prices are free-text strings, decoupled from Shopify's product catalog. This lets the
   stylist publish a Booksy-style menu without creating products.

2. **Booking/deposit product mode** (`main-product.liquid`). Any product **tagged with the
   booking tag** (`booking`, configurable) transforms its page into a deposit flow:
   - Shows a "deposit" tag next to the price and a configurable deposit note
     (overridable per product via metafield **`custom.deposit_note`**).
   - Optionally renders **Preferred date / Preferred time / Notes** fields
     (`show_request_fields`) saved as **line-item properties** (`properties[...]`) on the
     order — so the requested slot rides along with the order with no app.
   - Booking button label is configurable.
   - Supports **`@app` blocks**, so a real booking app (Cowlendar/Appointo) can drop its
     calendar widget in instead of the manual date/time fields.

3. **Metafields used:** `product.metafields.custom.deposit_note` (optional per-product
   deposit note override). No metaobjects.

4. **Global booking-URL fallback chain.** Most CTAs resolve a link in priority order:
   section/block-specific link → global `settings.booking_url` → contact form
   (`/pages/contact`). This is consistent across hero, header, about, service rows.

5. **Custom vanilla JS app** (`assets/theme.js`, no dependencies):
   - Mobile nav toggle.
   - Smooth-scroll for in-page `#anchor` links.
   - **Before/after comparison sliders** (range input drives a CSS `--pos` clip).
   - **Auto-injected carousel arrows** for any overflowing `.slider` (prev/next with
     scroll sync and disabled-state handling).
   - **Client-side variant picker** for `main-product`: matches selected options against a
     JSON variant blob to update variant id, price, and add-to-cart availability.
   - Re-initializes on `shopify:section:load` for the theme editor.

6. **Design tokens via Liquid → CSS custom properties** (`layout/theme.liquid`), consumed
   throughout `base.css`. Colors are editor-driven; fonts/width are hardcoded.

7. **Booking app embeds wired at the store level** (`settings_data.json` blocks): Appointo
   (enabled) and Cowlendar (disabled). These are app embeds, not theme code — manage them
   in Theme editor → App embeds.

## 8. Known gaps (hardcoded / placeholder content that probably should be addressed)

**Placeholder / incorrect data to replace before launch:**
- **Map embed points to "Hooters"** (`settings_data.json` → `map_embed`). Must be replaced
  with the real business location in Theme settings → Location & Hours.
- **Empty critical contact fields:** `phone`, `email`, `facebook_url`, and especially
  **`booking_url`** (the global fallback for every Book/Select button) are blank. Until
  `booking_url` is set, service/hero/about CTAs fall through to the contact form.
- **No logo uploaded** (`logo` empty) — header renders the business name as text instead.
- **Stale font settings** (`font_heading`/`font_body` = `assistant_n4`) in
  `settings_data.json` with no matching schema setting — leftover noise; safe to ignore or
  clean up.
- Seeded service prices, gallery images, and review text are **sample/placeholder content**
  and need to be replaced with the real menu, photos, and testimonials.

**Hardcoded values that arguably should be theme-editor-customizable but currently aren't:**
- **Typography is locked to Inter** (hardcoded in `layout/theme.liquid`); there is a
  Typography schema group but it only contains an explanatory paragraph — no font picker.
- **`--page-width` (1200px)** and **`--font-heading-weight` (600)** are hardcoded tokens.
- **404 page copy** (`main-404.liquid`) is hardcoded ("404" + message + "Back home").
- **Cart "Update" button label** and a few inline strings in `main-cart.liquid` are
  hardcoded rather than localized (most cart strings do go through `locales/en.default.json`).
- **Booking confirmation note** in `main-product.liquid` ("You'll pay the deposit at
  checkout… {{ shop.name }} will confirm…") is hardcoded.
- **Hero secondary-button fallback** (`/collections/services`) and **service-row fallback**
  (`/pages/contact`) are hardcoded route literals; fine as defaults but not surfaced as
  settings.
- **Footer "Hours" and "Find us" column headings** are hardcoded `<h4>` text in
  `footer.liquid` (only the links-column title is a setting).
- **Single locale** (`en.default.json`); no other languages.

None of these block functioning — they're the difference between "works" and "fully
self-serve in the editor."

## 9. Dependencies

- **No package manager / build tooling.** There is **no `package.json`**, no lockfile, no
  `node_modules`, no bundler, no task runner, no CI config (`.github/` absent). What's in
  the repo is exactly what ships.
- **No JS frameworks.** `assets/theme.js` is ~126 lines of dependency-free vanilla JS
  (IIFE, no imports).
- **No CSS preprocessor.** `assets/base.css` is plain CSS (~634 lines) consuming CSS custom
  properties injected by Liquid.
- **External runtime dependencies (loaded by the browser, not bundled):**
  - **Google Fonts — Inter** (`<link>` in `layout/theme.liquid`).
  - **Google Maps embed iframe** (merchant-provided HTML in theme settings).
  - Optional **Instagram embed** HTML (merchant-provided in the Reviews section).
- **Shopify app embeds (store-level, optional):** Appointo (enabled), Cowlendar booking
  (disabled) — referenced in `settings_data.json`.
- **Tooling for development/deploy:** Shopify CLI (`shopify theme dev` / `shopify theme
  push`) or the GitHub → Shopify theme connection. Nothing to `npm install`.

### Quick start for a new maintainer

1. Pull the repo, connect it to the Shopify store (GitHub theme connection or
   `shopify theme dev` with the store's CLI auth).
2. Edit Liquid/CSS/JS in the repo; edit content in the Shopify theme editor.
3. First priorities (from §8): set `booking_url`, real `map_embed`, `phone`/`email`,
   upload the `logo`, and replace seeded service/gallery/review content.
4. Read `README.md` for the operator setup checklist and the intake-field → settings
   mapping; read `BUILD_SPEC.md` if you need the original full build context.
